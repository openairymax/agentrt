# AgentOS 原子层 (Atoms)

**路径**: `agentos/atoms/`

Atoms 层是 AgentOS 的**核心系统层**，位于系统架构的最底层，提供智能体操作系统所需的基础运行时能力和原子化服务。Atoms 层遵循微内核设计原则，以最小化核心、最大化可扩展性为宗旨，为上层 Daemon 服务和应用程序提供稳定、高效的底层支撑。

---

## 架构定位

AgentOS 采用分层架构设计，Atoms 层位于最底层：

```
┌──────────────────────────────────────────────┐
│             Applications (OpenLab)            │
├──────────────────────────────────────────────┤
│             Ecosystem (Toolkit/SDK)           │
├──────────────────────────────────────────────┤
│              Daemon Services                  │
├──────────────────────────────────────────────┤
│          ★ Atoms (Core System Layer) ★       │
├──────────────────────────────────────────────┤
│            Operating System / Hardware        │
└──────────────────────────────────────────────┘
```

Atoms 层直接构建于操作系统之上，通过封装系统资源（CPU、内存、存储、网络）为上层提供统一的智能体运行时抽象。

---

## 核心模块

Atoms 层由以下七个核心模块组成：

### 1. CoreKern — 微内核

CoreKern 是所有原子的"核中之核"，提供最小化的内核服务集合。它遵循真正的微内核设计，仅包含 IPC/Binder、内存管理、任务调度、时间服务四个原子级能力。所有其他服务（文件系统、网络、设备驱动等）均以用户态服务的形式运行。

- **设计原则**: 最小特权、机制与策略分离、服务化
- **核心能力**: 进程/线程管理、内存分配、中断处理、时钟服务
- **通信机制**: Binder IPC（进程间通信）

[详细文档](corekern/README.md)

### 2. CoreLoopThree — 核心运行时

CoreLoopThree 是 AgentOS 的核心运行时引擎，采用创新的"三循环"架构：认知循环（Cognition Loop）、执行循环（Execution Loop）和学习循环（Learning Loop）。该设计受心理学"双系统理论"启发，实现了感知-思考-行动的闭环。

- **认知循环**: 环境感知、信息处理、决策生成
- **执行循环**: 任务执行、指令调度、状态反馈
- **学习循环**: 经验积累、模式发现、策略优化
- **状态管理**: 会话上下文、任务上下文、全局上下文、记忆上下文

[详细文档](coreloopthree/README.md)

### 3. Memory — 内置记忆子系统

Memory 是 AgentOS 的内置记忆底座，提供 L1 原始层（文件系统+SQLite）和 L2 特征层（FAISS 向量索引）两级记忆能力。作为免费开源组件，无需外部 MemoryRovol、FAISS 或 SQLite 依赖即可运行。

- **L1 原始层**: 基于文件系统和 SQLite 的原始数据存储
- **L2 特征层**: FAISS 向量索引，支持语义相似度搜索
- **零外部依赖**: 内置实现，开箱即用

*文档待补充*

### 4. MemoryRovol — 商业记忆桥接

MemoryRovol 是商业级记忆系统桥接，提供 L1-L4 全功能记忆层次。需要外部 MemoryRovol 仓库，通过 CMake 选项 `-DAGENTOS_WITH_MEMORYROVOL=ON` 激活。

- **L1 原始层**: 基于文件系统和 SQLite 的原始数据存储
- **L2 特征层**: FAISS 向量索引，支持语义相似度搜索
- **L3 结构层**: 图数据库，保存知识图谱和关联结构
- **L4 模式层**: 拓扑数据分析(TDA) + HDBSCAN 聚类，挖掘深层模式
- **检索策略**: 吸引子网络检索、分层级联检索、混合搜索

[已迁移至独立仓库](../../../MemoryRovol/README.md)

### 5. Syscall — 系统调用层

Syscall 层提供统一的系统调用抽象接口，屏蔽底层异构硬件的差异，为上层应用提供标准化的 API 调用机制。支持线程安全、可重入、跨语言 FFI 调用。

- **接口分类**: 任务调用、内存调用、会话调用、遥测调用、代理调用
- **安全机制**: 参数校验、权限检查、审计日志
- **调用模式**: 同步调用、异步回调、事件通知

[详细文档](syscall/README.md)

### 6. TaskFlow — 任务流引擎

TaskFlow 提供了 DAG（有向无环图）驱动的任务编排能力，支持复杂工作流的定义、调度、执行和监控。它与 CoreLoopThree 的执行循环紧密配合，提供更高级的任务编排原语。

- **生命周期**: 创建 → 调度 → 执行 → 完成
- **依赖管理**: DAG 依赖图，自动拓扑排序
- **优先级控制**: 5 级优先级队列
- **持久化**: SQLite 状态存储

[详细文档](taskflow/README.md)

### 7. Frameworks — 框架集成层

Frameworks 层采用适配器模式，为外部 AI 框架提供统一的集成接口。通过定义标准的"能力模型"，允许 LangChain、MCP、A2A、OpenAI 等框架无缝接入 AgentOS 生态。

- **适配器模式**: 每个框架封装为独立适配器
- **能力模型**: 13 标准能力定义
- **热插拔**: 运行时动态加载/卸载框架适配器

[详细文档](frameworks/README.md)

## 模块关系

```
                    ┌─────────────┐
                    │  CoreKern   │
                    │  (微内核)    │
                    └──────┬──────┘
                           │
           ┌───────────────┼───────────────┐
           │               │               │
           ▼               ▼               ▼
    ┌───────────┐   ┌───────────┐   ┌───────────┐
    │ Syscall   │   │ TaskFlow  │   │ CoreLoop  │
    │ (系统调用) │◄──┤ (任务流)   │◄──┤ Three     │
    └───────────┘   └───────────┘   │ (运行时)   │
                                    └──────┬────┘
           ┌───────────┐                   │
           │ Memory    │◄──────────────────┘
           │ (内置记忆) │
           └─────┬─────┘
                 │ (桥接)
           ┌───────────┐
           │ Memory    │
           │ Rovol     │
           │ (商业桥接) │
           └───────────┘

    ┌──────────────────────────────────────┐
    │            Frameworks                │
    │  (LangChain / MCP / A2A / OpenAI)   │
    └──────────────────────────────────────┘
```

- CoreKern 为所有模块提供基础内核服务（IPC、内存、调度）
- Syscall 基于 CoreKern 提供统一的系统调用接口
- CoreLoopThree 使用 Syscall 和 TaskFlow 执行任务，使用 Memory 管理记忆
- Memory 提供内置 L1+L2 记忆能力，MemoryRovol 通过桥接扩展至 L1-L4
- TaskFlow 在 CoreKern 调度之上提供高级任务编排
- Frameworks 通过适配器接入各模块的能力

## 设计原则

- **微内核**: 最小化核心，最大化服务化
- **确定性**: 确定性的调度和资源管理
- **可组合**: 模块间通过明确定义的接口组合
- **无锁设计**: 尽量减少锁竞争，使用无锁数据结构
- **零拷贝**: 核心数据路径上的零拷贝设计
- **异步优先**: 非阻塞 I/O 和事件驱动
