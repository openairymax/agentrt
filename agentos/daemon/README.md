# Daemon — 用户态服务层

`agentos/daemon/` 是 AgentOS 的用户态服务层，由多个独立进程的守护进程组成，为智能体提供完整的后端服务支持。

## 设计目标

- **服务化架构**：每个守护进程独立运行，通过 IPC 通信协作
- **职责单一**：每个守护进程只负责一个核心领域
- **可插拔**：守护进程可独立部署、升级和扩缩容
- **高可用**：支持主备切换、故障恢复和负载均衡

## 核心守护进程

| 守护进程 | 目录 | 职责 |
|----------|------|------|
| **API 网关** | `gateway_d/` | 外部请求接入，协议转换与路由 |
| **LLM 服务** | `llm_d/` | 大语言模型调用与缓存 |
| **工具执行** | `tool_d/` | 外部工具注册与安全执行 |
| **任务调度** | `sched_d/` | 任务分发与调度策略 |
| **应用市场** | `market_d/` | Agent/Skill 市场管理 |
| **监控告警** | `monit_d/` | 指标采集与告警管理 |
| **通道服务** | `channel_d/` | 通信通道管理与消息路由 |
| **信息服务** | `info_d/` | 系统信息查询与状态报告 |
| **通知服务** | `notify_d/` | 多渠道通知推送（邮件/Slack/Discord） |
| **观测服务** | `observe_d/` | OpenTelemetry 可观测性数据采集 |
| **公共服务** | `common/` | 共享工具库与兼容层 |

## 守护进程间通信

所有守护进程通过统一的 JSON-RPC 2.0 协议通信：

| 通信方式 | 适用场景 | 性能 |
|----------|----------|------|
| Unix Socket | 同机守护进程 | < 100μs |
| TCP | 跨机守护进程 | < 1ms |
| 共享内存 | 高性能数据交换 | < 10μs |

## 守护进程生命周期

```
INIT → CONFIG_LOAD → SERVICE_REGISTER → IDLE → BUSY → SHUTDOWN
  ↓        ↓             ↓               ↓      ↓        ↓
初始化   加载配置    注册到服务发现     等待    处理    优雅关闭
```

## 启动方式

```bash
# 启动单个守护进程
./gateway_d --config gateway_config.json

# 使用管理器启动所有守护进程
./daemon_manager --start-all

# 查看守护进程状态
./daemon_manager --status
```
